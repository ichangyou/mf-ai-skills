# 沐风月报生成器

## Overview

根据过去一个月的对话记录，自动生成结构化月报，并给出月度亮点分析、数据总结与下月方向建议。

## Process

### Step 1：确定月度周期

计算本次月报所属的自然月，格式为：
`2026年XX月 - Claude月报（MM.01 - MM.DD）`

使用今天的日期确定当月的起止日期（1日到最后一天）。

### Step 2：回顾对话，提取信息

通读本月所有 session 的对话内容，识别并归类以下信息。月报要求比周报更加深入，需要总结趋势和规律：

| 类别 | 关键词/信号 | 月报关注点 |
|------|------------|----------|
| 技术成长 | 写代码、debug、架构、工具、技术方案 | 掌握了哪些新技能、完成了哪些项目 |
| 学习 | 读书、看文章、学新知识、研究 | 读完了哪些书、学了哪些系统性知识 |
| 写作 | 写文章、公众号、博客、月报 | 发布了多少篇内容、哪篇反响最好 |
| 身体健康 | 运动、锻炼、睡眠、饮食 | 运动频率和规律性如何 |
| 生活探索 | 新体验、餐厅、旅行、兴趣爱好 | 有哪些值得记录的新体验 |
| 想法/反思 | 总结、思考、复盘、计划 | 本月最重要的认知升级是什么 |
| 目标追踪 | 计划、目标、完成、未完成 | 月初设定的目标完成情况如何 |

**如果某类别无内容，保留标题但标注"本月暂无记录"，不要删除该类别。**

### Step 3：生成月报

严格按照以下模板输出，每个 emoji 和格式都要保留：

```
📅 2026年XX月 - Claude月报（MM.01 - MM.DD）

🏆 本月最重要的 3 件事：
1. [动词开头，简洁有力]
2. [动词开头，简洁有力]
3. [动词开头，简洁有力]

📌 技术成长
[本月完成了哪些项目、掌握了哪些技能、解决了哪些技术难题；如无则写"本月暂无记录"]

📚 学习
[读了哪些书、学了哪些课程、有哪些系统性收获；如无则写"本月暂无记录"]

📝 写作
[发布了哪些文章、总计多少篇、哪篇反响最好；如无则写"本月暂无记录"]

🧘 身体健康
[本月运动规律性、睡眠质量、饮食状态总体评价；如无则写"本月暂无记录"]

☕ 生活探索
[本月值得记录的新体验、旅行、餐厅、兴趣爱好；如无则写"本月暂无记录"]

🎯 目标追踪
[月初目标的完成情况，哪些达成，哪些未达成，原因是什么；如无则写"本月暂无记录"]

🧠 本月最重要的认知升级：
[1-3 条本月最值得记录的思维转变或深刻感悟]
```

### Step 4：月度深度分析

月报正文之后，紧接着输出以下分析（比周报更深入，关注趋势和模式）：

---

**✨ 本月做得好的地方**

列出 3-5 条亮点，聚焦于月度维度的成就，说明为什么值得肯定。

**⚠️ 本月做得不好的地方**

列出 2-4 条不足，诚实直接，重点指出在整个月中反复出现的问题模式。

**📊 本月规律与趋势**

基于整个月的对话，分析出 2-3 条行为规律（如：工作效率高峰期、容易放弃的类型任务、注意力容易分散的场景等）。

**💡 下月行动建议**

针对不足和规律，给出 3 条下月可以立刻执行的具体改进计划，要有明确的执行标准（如频率、数量、截止时间）。

**🔭 下月展望**

简短写出下月 1-2 个最重要的目标方向（不超过 50 字）。

---

## Output Rules

- 月报内容用**中文**输出
- 月报比周报更注重**趋势和规律**，不只是罗列事件
- 亮点/不足/建议要基于对话事实，不要凭空捏造
- 如果对话内容不足以支撑某类别，如实说明，不要编造
- 语气：真实、坦诚、鼓励为主，批评要建设性
- 月报正文总字数控制在 800-1200 字（不含分析部分）
- 分析部分要比周报更深入，聚焦规律而非孤立事件

## Common Mistakes

| 错误 | 修正 |
|------|------|
| 删除没有内容的类别 | 保留类别，填写"本月暂无记录" |
| 分析部分只说好话 | 不足要诚实写出来，否则没有复盘价值 |
| 改进建议太抽象 | 必须是下月可立刻执行的具体动作，含标准 |
| 月报写成多个周报拼接 | 月报要提炼趋势和规律，不是周报的简单叠加 |
| 忽略目标追踪模块 | 月度复盘必须回顾目标达成情况 |
| 月份计算错误 | 用今天日期确认当月起止日期 |
