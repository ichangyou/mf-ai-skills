# Article Templates

## Standard Technical Article
```markdown
# [标题] - [核心关键词]

## 引言
- 问题背景或场景描述
- 为什么值得关注
- 本文将覆盖的内容

## 正文
### 问题分析
- 问题现象
- 根因分析

### 解决方案
- 方案概述
- 实现步骤
- 代码示例（可运行）

### 最佳实践
- 实战经验
- 常见陷阱与避坑
- 性能或可维护性建议

## 总结
- 核心要点回顾
- 延伸阅读或相关资源

## 参考资料
- 官方文档
- 相关文章或讨论
```

## Tutorial Article
```markdown
# [教程标题] - 从零到一实现 XXX

## 你将学到什么
- 列出 3-5 个关键收获

## 前置要求
- 基础知识
- 开发环境

## 步骤 1: [第一步]
- 说明
- 代码示例
- 运行结果

## 步骤 2: [第二步]
...

## 完整代码
- 可运行的完整代码
- GitHub 仓库链接（如有）

## 下一步
- 功能扩展建议
- 进阶学习方向
```

## Problem-Solving Article
```markdown
# [标题] - [问题 + 解决方向]

## 问题场景
- 具体场景
- 复现步骤
- 影响范围

## 问题分析
- 根因定位
- 关键证据

## 解决方案
- 方案对比
- 最终选择与原因
- 实现细节

## 最佳实践
- 预防措施
- 常见坑

## 总结
- 关键结论
- 后续行动
```

## Reusable Snippets

### 引言模板
```markdown
最近在做 [项目名称] 的时候遇到了一个问题：[简述问题]。很多使用 [技术栈] 的同学可能也会遇到类似情况。

这篇文章会分享：
- 问题的根本原因
- 几种方案的对比
- 实际项目中的最佳实践
```

### 问题描述模板
```markdown
在 [具体场景] 中，需要 [需求描述]。直接使用 [某种方法] 会出现：
1. [问题 1]
2. [问题 2]

运行后出现 [错误或异常]，原因是 [根因分析]。
```

### 方案对比模板
```markdown
| 方案 | 优点 | 缺点 | 适用场景 |
|---|---|---|---|
| 方案 A | 简单易用 | 灵活性差 | 小型项目 |
| 方案 B | 功能强大 | 学习成本高 | 中大型项目 |
| 方案 C | 折中方案 | 需要额外配置 | 通用场景 |

我选择 **方案 B**，原因是：[详细说明]
```

### 总结模板
```markdown
本文介绍了 [核心内容]：
- 关键点 1
- 关键点 2
- 关键点 3

建议在实际开发中：
1. [建议 1]
2. [建议 2]
3. [建议 3]
```
