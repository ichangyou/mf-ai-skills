# 沐风周报生成器

## Overview

根据过去一周的对话记录，自动生成结构化周报，并给出亮点分析与改进建议。

## Process

### Step 1：确定周期

计算本次周报所属的自然周（周一到周日），格式为：
`2026年第XX周 - Claude周报（MM.DD - MM.DD）`

使用今天的日期往前推，确定本周的起止日期。

### Step 2：回顾对话，提取信息

通读本次 session 以及近期 session 的对话内容，识别并归类以下信息：

| 类别 | 关键词/信号 |
|------|------------|
| 技术成长 | 写代码、debug、架构、工具、技术方案 |
| 学习 | 读书、看文章、学新知识、研究 |
| 写作 | 写文章、公众号、博客、周报 |
| 身体健康 | 运动、锻炼、睡眠、饮食 |
| 生活探索 | 新体验、餐厅、旅行、兴趣爱好 |
| 想法/反思 | 总结、思考、复盘、计划 |

**如果某类别无内容，保留标题但标注"本周暂无记录"，不要删除该类别。**

### Step 3：生成周报

严格按照以下模板输出，每个 emoji 和格式都要保留：

```
📅 2026年第XX周 - Claude周报（MM.DD - MM.DD）

✅ 本周最关键任务：
[列出 1-3 件本周最重要的事，用动词开头，简洁有力]

📌 技术成长
[具体写了什么、做了什么、解决了什么问题；如无则写"本周暂无记录"]

📚 学习
[读了什么、学了什么新内容；如无则写"本周暂无记录"]

📝 写作
[写了什么文章或内容；如无则写"本周暂无记录"]

🧘 身体健康
[运动、睡眠等情况；如无则写"本周暂无记录"]

☕ 生活探索
[生活中的新体验；如无则写"本周暂无记录"]

🧠 想法/反思：
[本周最值得记录的思考或感悟，1-3 条]
```

### Step 4：亮点分析与改进建议

周报正文之后，紧接着输出以下分析：

---

**✨ 本周做得好的地方**

列出 2-4 条亮点，具体说明为什么值得肯定（不要泛泛而谈）。

**⚠️ 本周做得不好的地方**

列出 2-3 条不足，诚实直接，不要回避。

**💡 改进建议**

针对每一条不足，给出 1 条具体可操作的建议，下周可以立刻执行的那种。

---

## Output Rules

- 周报内容用**中文**输出
- 亮点/不足/建议要基于对话事实，不要凭空捏造
- 如果对话内容不足以支撑某类别，如实说明，不要编造
- 语气：真实、坦诚、鼓励为主，批评要建设性
- 总字数控制在 500-800 字（不含分析部分）

## Common Mistakes

| 错误 | 修正 |
|------|------|
| 删除没有内容的类别 | 保留类别，填写"本周暂无记录" |
| 分析部分只说好话 | 不足要诚实写出来，否则没有复盘价值 |
| 改进建议太抽象 | 必须是下周可立刻执行的具体动作 |
| 周期计算错误 | 用今天日期确认周一到周日的范围 |
